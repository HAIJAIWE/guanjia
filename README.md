# guanjia（管家）系列技能

> 你工具有一堆 Agent、Skill、MCP，但不知道什么时候该用哪个？管家帮你统一调度。

## 为什么用管家

AI 编码工具（Trae、Cursor、Claude Code 等）装了一堆技能：代码搜索、后端开发、前端设计、API 测试……但每次你提需求，AI 不知道该调哪个，经常用错工具或者漏掉该用的。

管家就是解决这个问题的：

1. 你提需求 → 管家扫描当前所有可用的工具
2. 逐个分析哪个工具最适合你的任务，打分排序
3. 弹窗列出最合适的 top 10，让你勾选确认
4. 按顺序串行调用，前一个的输出传给后一个

你不用手动指定工具，也不用担心用错了。

## 目录结构

```

├── zh/                          ← 中文版
│   ├── guanjia-auto/SKILL.md    ─ 主动接管（自动触发，不退场）
│   └── guanjia-passive/SKILL.md ─ 被动接管（说"管家"才触发）
├── en/                          ← 英文版
│   ├── guanjia-auto/SKILL.md
│   └── guanjia-passive/SKILL.md
├── README.md
└── README.en.md
```

## 版本选择

| 版本                          | 触发方式     | 行为               |
| --------------------------- | -------- | ---------------- |
| **主动版** (`guanjia-auto`)    | 自动接管每条消息 | 检测→匹配→弹窗→调用，不退场  |
| **被动版** (`guanjia-passive`) | 说「管家」才触发 | 检测→匹配→弹窗→调用，跑完就退 |

选一个版本安装即可，**不要两个都装**。

> ⚠️ auto 版是"所有消息自动触发"，passive 版是"说'管家'才触发"。两个同时装会导致冲突：当你说"管家"时两个技能同时触发，指令互相矛盾（一个说不退场、一个说干完就退），AI 会不知道听谁的。

***

## 安装

选好版本，把对应文件夹放到工具的 skills 目录：

```
你的项目/
└── .trae/skills/
    ├── guanjia-auto/      ← 或用 guanjia-passive
    │   └── SKILL.md
    └── ...
```

opencode 则放在 `~/.claude/skills/<版本名>/SKILL.md`。

## 使用方法

### 主动版

安装即生效，正常说话即可：

```
你：帮我搜索一下最新的AI资讯       ← 管家自动调度
你：写一个登录页面                 ← 管家自动调度
你：测试一下API                   ← 管家自动调度
```

说 `退出管家` 或 `解除接管` 可退出。

> ⚠️ 主动版每条消息都会触发，如果不需要全程自动调度，建议用被动版。

### 被动版

只有喊「管家」才出来，不影响正常对话：

```
你：帮我写个登录页面               ← 正常对话，管家不干预
你：管家，帮我检测一下有哪些工具可用  ← 管家触发
你：继续                          ← 继续上次分配
你：谢谢                          ← 正常对话，管家不干预
```

干完活就退，下次再用再喊。

### 中断恢复

如果网络波动或意外中断，重新连接后：

- **主动版**：检测到上次任务未完成则直接继续，不重新调度
- **被动版**：重新说「管家」即可触发

## 内部原理

1. **事前检测** — 扫描所有 Agent、Skill、MCP 工具，构建能力清单（排除自身）
2. **意图分类** — 分析用户任务属于哪类（编码/搜索/设计/测试 等）
3. **语义匹配打分** — 三步语义分析：读懂用户任务 → 读懂能力描述 → 语义匹配判断。杜绝关键词陷阱（如"设计"≠前端设计、"API"≠后端架构）
4. **弹窗选择** — 列出 top 10 让用户多选勾选，记住原始任务不丢失
5. **按序调用** — 按用户勾选顺序串行执行，原始任务完整传递，前一个输出传给后一个
