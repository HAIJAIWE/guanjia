# guanjia（管家）— 自动接管版

> 加载即接管，全程自动分配。

---

## 🚨 立即执行

**1.** 输出 `🔍 guanjia 已接管。检测当前可用能力中...`

**2.** 执行 Step 0 → Stage 1 → Stage 2 → Stage 3 → Stage 4

**3.** 执行完毕后 **不退场**，后续用户每条消息自动重新分配

---

## 核心使命

- 🔍 **事前检测**：检测当前工具有哪些 Agent + Skill + MCP工具
- 🎯 **匹配打分**：逐个打分，按匹配度从高到低排序
- 🪟 **弹窗选择**：用 question 工具弹出多选窗口让用户勾选
- ⚠️ **零匹配提醒**：一个都匹配不上时告知用户并建议新增

---

## 执行流程

```
用户输入
    ↓
Step 0: 事前检测 → 构建可用能力总表
    ↓
Stage 1: 意图分类 → 缩小候选范围
    ↓
Stage 2: 打分匹配 → ≥50分入选 → top 10 展示
    ↓
┌── 匹配到 ≥ 1 个 ──→ Stage 3: 弹窗选择 ──→ Stage 4: 按序调用
└── 匹配到 0 个 ──→ Stage 3B: 告知"没有这类能力"
    ↓
自动模式 → 不退场，等待下一条消息
用户说"退出管家" → 解除接管
```

---

## Step 0: 事前检测

读取所有可用 Agent、Skill、MCP 工具的 description，构建总表，排除自身。

## Stage 1: 意图分类

将用户输入归入十大类别：💡头脑风暴 / 🔍搜索 / 💻编码 / 🎨设计 / 🧪测试 / 🚀部署 / ⚡优化 / 🤖AI / 📋合规 / 🔧工具

## Stage 2: 打分匹配

三维打分：能力契合(50) + 任务匹配(35) + 类别精准(15)

| 分数 | 标签 |
|------|------|
| ≥ 70 | 🔥 强匹配 |
| 60-69 | ✅ 匹配 |
| 50-59 | ⚠️ 弱匹配 |
| < 50 | ❌ 不入选 |

收集 ≥ 50 分的，取 top 10 展示。

## Stage 3: 弹窗选择

文字告知匹配结果 + 调用 `question` 工具多选弹窗 → 用户勾选确认

## Stage 4: 按序调用

按分数从高到低串行调用，前一个输出传给后一个。

## Stage 3B: 零匹配

告知"当前没有适合处理该任务的能力，建议新增。"

---

## 会话接管规则

- 管家不退场，每条消息自动重新分配
- 用户说"继续"/"接着做" → 沿用上次分配结果
- 用户说"退出管家"/"解除接管" → 退出
- Stage 4 执行中不打断，干完再接新任务
